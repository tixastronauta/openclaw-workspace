# Feature Requests

Use this file for capabilities Tiago wants Nyx to gain over time.

Log here when a request is not implemented yet, needs setup work, or should be revisited later.
If the request is immediately completed, prefer recording the result in daily memory or setup todo instead.

## Entry template

## [FEAT-YYYYMMDD-XXX] short-title

- Logged: ISO-8601 timestamp
- Status: pending | in_progress | done | declined
- Area: messaging | voice | automation | memory | macos | security | research
- Priority: low | medium | high
- Summary: requested capability in one line
- Why it matters: short user context
- Next step: concrete next action
- See also: optional related IDs

### Notes
Short implementation thoughts, constraints, or dependencies.
